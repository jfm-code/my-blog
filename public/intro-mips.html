Some basics stuff to know

Overall, a mips program consists of 2 main sections: data section and text section

1).data (also called the data section) ⇒ this is used to declare and initialize data elements, such as variables, constants, and strings.

2) .text ⇒ (also called the text section) ⇒ this is used to contain the program's instructions.

Moreover, we can comment using the symbol #

Example:

.data
my_variable:   .word 42
my_string:     .asciiz "Hello, MIPS!"

.text
main:
    # Your MIPS instructions go here

More about data section

We use this section to declare variables, including integers, string, arrays,…

  • How to declare an integer:

Step 1: have a label (in this case it is “my_variable”)

Step 2: use the .word directive (it is usually used for integers)

Step 3: put an integer next to it to initialize the value 

.data
my_variable:   .word 42
  • How to declare a string:

Step 1: have a label (in this case it is “my_string”)

Step 2: use the .asciiz directive to get the null terminator for string

Step 3: put a string next to it

.data
my_string:     .asciiz "Hello, MIPS!"

More about text section

  • The “main” thing:

main: marks the entry point of the program, and the program's execution begins at that label. It's common to perform various tasks in the main section, and when you're done, you can use system calls (like li $v0, 10 and syscall) to exit the program.

In this case we use $v0 and 10 because $v0 = 10 is the system call number for program exit.

.text
.globl main

main:
    # MIPS assembly instructions go here

    # Exit the program
    li $v0, 10
    syscall

The registers

  • The register $v:

$v0 = 10 is the system call number for program exit.

$v0 = 1 is the system call number for printing an integer.

$v0 = 4 is the system call number for printing a string.

$v0 = 5 is used for reading an integer.

$v0 and $v1 is used for input and output

Reading a string is much more complex to using just one register

  • The register $a:

$a0 is often used for the first argument.

$a1 is often used for the second argument.

  • $a0 (argument register 0) is typically used for passing the address of the buffer where you want to store the input or output of the system call.
  • $a1 (argument register 1) is often used for passing additional information related to the system call, such as the length of the buffer.

These are argument registers for general use. They are used to pass arguments to functions or system calls.

  • The register $t:

$t0 → $t9 : are for general purpose. They are also known as temporary registers

  • The register $s:

$s0 → $s7 : are for general purpose. They are also known as save registers

  • Other register:

$sp : stack pointer register

$ra : return address register

⇒ the purpose of jr $ra is to ensure that execution resumes in the main program after the point where the function was called, allowing the main program to continue running after the function has finished its work.

Data Transfer (Load & Store)

MIPS has two groups of data transfer instructions: ‘load’ to store data in a register and ‘store’ to
save register data in a memory location.

Load (reg (register) → memory)

  • Load immediate (li) ⇒ used to load a constant
  • Load byte (lb) ⇒ used to load a byte of data from memory (fetch a single byte from memory to store in a register)
  • Load word (lw) ⇒ used to load more than a byte of data from memory (fetch a word (4 bytes) from memory to store in a register)
  • Load address (la) ⇒ used to load the address of something into the arguments

Store (memory → reg (register))

  • Store word (sw) ⇒ used to store data into memory
  • Store byte (sb) ⇒ used to store byte into memory

Addressing

Attention: MARS can handle X+4, but cannot handle X-4

There are 3 ways to address a position:

  • Direct

– used when memory address is specified (for example arrayX+4 is the address)

lw  $t1, arrayX+4 
#we can load the value from address arrayX+4 here
  • Indirect

– the address is loaded to a register first, then we access that address through the register

la  $t2, arrayX+4 
lw  $t1, ($t2)
#in this case we load the address arrayX+4 to the register first 
#then we load the word using that address stored in $t2
  • Indexed

– There will be a register specifies start of memory block (the one in the braces)

– And then there will be a constant specifies the offset (the displacement from the start of memory block which is written BEFORE the braces

lw  $t1, 8($t2)
#for example in this case 8($t2) means $t2 + 8 

li  $s0, 4
lw  $t3, T + 4($s0) 
#this example is more complicated. First we have a constant load into $s0
#then we try to access the address using $s0. The 4($s0) here means $s0 + 4
#but we also know that $s0 = 4 so $s0 + 4 = 4 + 4
#therefore T + 4($s0) is equal to T + 4 + 4 = T + 8
#another example from the lab 
	.data
array: .word 0x37, 0x55, 0xF
	.text
			la $s0, array      # $s0 = 0x1100
			la $s1, array+4    # $s1 = 0x1104
			la $s2, 8($s0)     # $s2 = 0x1108, this one is $s0+8
			li $a0, 4          # $a0 = 4
			lw $t0, 0($s0)     # $t0 = 0x37, this one is $s0+0
			lw $t3, array($a0) # $t3 = 0x55, this one is array+$a0 = array+4
			lw $t1, 8($s0)     # $t1 = 0xF
			lb $t2, ($s0)      # $t2 = 0x37

I/O Instructions

When we want to print AN INTEGER, the steps are:

  1. move $a0, $s3 #move the value to register $a0 first, remember it can ONLY be $a0
  1. li $v0, 1
  1. syscall

When we want to print A STRING, the steps are:

  1. la $a0, name_of_string
  1. li $v0, 4
  1. syscall

Branch Instructions

  • b [label]

    ⇒ this is used just like goto

    • "b" stands for branch.
    • "label" is the target label or the address to which you want to jump unconditionally.
    • for example: b loop2 means goto loop2
  • beq $s, $t, [offset/label]

    ⇒ this is used to compare

    • “beq” stands for branch if equal
    • for example: beq $s, $t, label would branch to label if the values in registers $s and $t are equal.
  • bne $s, $t, [offset/label]

    ⇒ this is also used to compare

    • “bne” stands for branch if not equal
    • for example: bne $s, $t, label would branch to label if the values in registers $s and $t are not equal.
  • beqz $s, [label]

    ⇒ “branch if equal to zero”

    • if $s is equal to 0 then branch to [label]
  • bge $t0, $t1, [label]

    ⇒ “branch if greater than or equal”

    • if $t0 is greater than or equal $t1 then branch to [label]
  • ble $t0, $t, [label]

    ⇒ “branch if less than or equal”

    • if $t0 is less than or equal $t1 then branch to [label]
  • bgt $t0, $t, [label]

    ⇒ “branch if greater than”

    • if $t0 is greater than $t1 then branch to [label]
  • blt $t0, $t, [label]

    ⇒ “branch if less than”

    • if $t0 is less than $t1 then branch to [label]
